## 📂 Curriculum Structure

Each folder contains practical implementations, code samples, and a dedicated `README.md` detailing the technical depth of the sprint.

### Phase 1: JavaScript Mastery & Fundamentals
*   **[Sprint 01: JS for React](./sprint-01)** – ES6+ features, Functional Programming (Pure functions, Composition), and Array methods.
*   **[Sprint 02: OOD & OOP Patterns](./sprint-02)** – Classes, Inheritance, Prototypes, and Context management (`call/apply/bind`).
*   **[Sprint 03: Asynchronous JS & Closures](./sprint-03)** – Event Loop, Promises, Async/Await, Generators, and Lexical scoping.

### Phase 2: React Core & Ecosystem
*   **[Sprint 04: Declarative UI & Styling](./sprint-04)** – JSX Architecture and modern CSS-in-JS/Styling strategies.
*   **[Sprint 05: React Lifecycle & Logic](./sprint-05)** – State management, Props, Event handling, and fundamental Hooks.
*   **[Sprint 06: Type-Safe Forms](./sprint-06)** – Implementation of Zod and React Hook Form for robust data validation.
*   **[Sprint 07: Browser APIs & Workers](./sprint-07)** – Local/Session Storage, WebWorkers for multi-threading, and WebSockets.
*   **[Sprint 08: Tools](./sprint-08)** – Webpack, Babel, ESLint, and an introduction to Next.js.
*   **[Sprint 09: Backend Integration](./sprint-09)** – Node.js, Express, and efficient HTTP communication with Axios.

### Phase 3: Advanced Architecture & Scaling
*   **[Sprint 10: Routing Architecture](./sprint-10)** – Deep dive into React Router 6 (Nested routes) and Next.js File-based routing.
*   **[Sprint 11: State Management (Redux Toolkit)](./sprint-11)** – Scalable state management using modern Redux practices.
*   **[Sprint 12: Internationalization](./sprint-12)** – Implementing i18n for multi-language support.
*   **[Sprint 13: TypeScript Integration](./sprint-13)** – Static typing for React components, state, and event handlers.
*   **[Sprint 14: Advanced React Patterns](./sprint-14)** – Refs, Higher-Order Components (HOCs), and component abstraction.

### Phase 4: Performance, Testing & UI Systems
*   **[Sprint 15: Optimization & Context](./sprint-15)** – Memoization (`useMemo`, `useCallback`), Context API, and Virtualization.
*   **[Sprint 16: Quality Assurance](./sprint-16)** – Unit & Integration testing using Jest and React Testing Library (RTL).
*   **[Sprint 17: Design Systems](./sprint-17)** – Building enterprise-grade UIs with Material UI (MUI) and custom themes.
*   **[Sprint 18: Resilience & Reusability](./sprint-18)** – Error Boundaries and building sophisticated Custom Hooks.
